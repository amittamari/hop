# Specs

This directory contains durable product and design specs for `hop`. Treat these
as decision records for intended behavior, not as a replacement for the current
project contract in `docs/PROJECT.md` or the implementation boundaries in
`docs/ARCHITECTURE.md`.

## Current Specs

- `2026-06-05-product-spec.md`: Current product spec for the terminal search and
  resume experience.

## Maintenance

- Prefer dated specs for product or design decisions that explain why behavior
  exists.
- Keep `docs/PROJECT.md` updated with the current contract when a spec changes
  user-visible scope.
- Keep `docs/ARCHITECTURE.md` updated when a spec changes stable boundaries,
  invariants, or pressure points.
